## 長文の自動分割

再生モード (= `-o` を指定しない) では、**1500 文字を超える入力は文区切りで自動分割** され、順次 `say` に渡されます。これは長文での `say` の安定性と Ctrl-C の応答性を確保するためです。

- 分割記号: `。` `．` `.` `!` `?` `！` `？` 改行
- 各チャンクは 1500 文字に達してから、次の分割記号までを 1 つの塊として切り出す

### 例

```bash
cat long_article.txt | jmactts -L ja
```

5000 文字程度の記事なら 3〜5 個のチャンクに分割され、順番に再生されます。チャンクの境目では一瞬無音が入ります。

### ファイル出力時は分割しない

`-o` でファイル出力する場合は **分割せず** に全文を `say` へ渡し、単一の音声ファイルを生成します (分割すると複数ファイルになってしまうため)。

## Ctrl-C による中断

再生中に `Ctrl-C` (SIGINT) を押すと、現在再生中のチャンクを即座に停止し、後続のチャンクには進みません。

- 終了コード: **130** (= 128 + SIGINT)
- `SIGTERM` でも同様の挙動

### 仕組み

- `signal.NotifyContext` で生成した `context.Context` を `exec.CommandContext` に渡し、シグナル受信時に `say` プロセスを kill
- チャンク再生ループの各反復で `ctx.Err()` を確認し、キャンセル時は次のチャンクへ進まない

これにより、長文再生中でも 1 秒以内に停止します。
