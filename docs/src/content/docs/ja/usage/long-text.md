description: 長文の自動分割再生と Ctrl-C による即時停止

## 長文の自動分割

1500 文字を超える入力 (再生時のみ) は、文の区切りで分割して順次 `say` に渡されます。

- 分割記号: `。` `．` `.` `!` `?` `！` `？` 改行
- 各チャンクの長さは目安として 1500 ルーン以上で次の文区切りまで含める

### 例

```bash
cat long_article.txt | jmactts -L ja
```

5000 文字の記事であれば 3〜4 チャンクに分割され、順次再生されます。

### ファイル出力時の挙動

`-o` でファイル出力する場合は **分割しません**。`say` に全文をまとめて渡し、単一の音声ファイルを生成します。

## Ctrl-C による中断

再生中に `Ctrl-C` (`SIGINT`) を押すと、現在のチャンクの `say` プロセスを即座に停止し、後続のチャンクには進みません。

- 終了コード: **130** (= 128 + SIGINT)
- `SIGTERM` でも同様に動作します

### 仕組み

- `signal.NotifyContext` で生成した `context.Context` を `exec.CommandContext` に渡し、シグナル受信時に `say` を kill
- チャンク再生ループの各反復で `ctx.Err()` を確認し、キャンセル時は次へ進まない

これにより、長文再生中でも 1 秒以内に停止します。
